# dev-docs

A pack of skills for producing the written artifacts developers create alongside code: estimates, architecture decisions, and pull request descriptions.

---

## Contents

### Skills

#### `estimation-helper`

Breaks down a feature or user story into concrete tasks, assigns story point estimates, and produces a structured estimate document with risks, assumptions, and out-of-scope items.

**What it helps you do:**

- Decompose vague feature requests into actionable, estimable tasks
- Assign Fibonacci story points with reasoning
- Produce a shareable estimate document
- Surface risks and assumptions that stakeholders need to sign off on

---

#### `adr-writer`

Writes a clean Architecture Decision Record (ADR) in MADR format from a description of the problem, the options you considered, and the decision you made.

**What it helps you do:**

- Capture and communicate architecture decisions clearly
- Structure options with pros, cons, and trade-offs
- Produce a ready-to-commit ADR Markdown file

---

#### `pr-description-writer`

Generates a professional, well-structured pull request description from a git diff, a list of commits, or a plain-language description of what changed.

**What it helps you do:**

- Write PR descriptions that communicate intent, not just mechanics
- Include testing evidence, breaking changes, and follow-up items
- Adapt tone from a quick internal write-up to something more polished
